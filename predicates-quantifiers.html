Predicates & Quantifiers

1. Introduction to Predicates

Consider the statement: x + y = 10. Is it true or false? We cannot know until we assign values to the variables (parameters). This is a proposition with "gaps".

Definition: A predicate is a propositional function. It takes one or more objects as input and returns a proposition (which can then be evaluated as True or False).

Example: Let P(x) denote the statement "x > 3".
P(4) is True (because 4 > 3).
P(2) is False (because 2 is not > 3).

Multiple Parameters

Predicates can have multiple parameters.
Let Q(x, y) denote "x = y + 3".

  • Q(5, 2) is True (5 = 2 + 3).
  • Q(3, 0) is True (3 = 0 + 3).

2. Quantifiers (∀ and ∃)

Instead of plugging in specific values, we can "quantify" objects across a whole domain (e.g., all real numbers, all students). Quantification expresses the extent to which a predicate is true over a quantity of elements.

Universal Quantifier (∀)

Applies to all members of a specified set. Read as "for all" or "for every".

∀x P(x) means: For every value of x in the domain, P(x) is true.

Existential Quantifier (∃)

Applies to at least one member of a specified set. Read as "there exists" or "for some".

∃x Q(x) means: For some value of x in the domain, Q(x) is true.

Logical Equivalences with Quantifiers

  • ∀x (P(x) ∧ Q(x)) ≡ ∀x P(x) ∧ ∀x Q(x)
    "Every CS student learns programming AND discrete math"
  • ∃x (P(x) ∨ Q(x)) ≡ ∃x P(x) ∨ ∃x Q(x)
    "Some CS student excels in hardware OR software"

3. De Morgan’s Laws for Quantifiers

When you negate a quantifier, the quantifier flips (∀ becomes ∃, and vice versa), and you negate the predicate inside.

Negating Universal

¬(∀x E(x)) ≡ ∃x ¬E(x)

  • Statement: Everyone is evil.
  • Negation: Not everyone is evil.
  • Equivalent to: Someone is good (not evil).

Negating Existential

¬(∃x R(x)) ≡ ∀x ¬R(x)

  • Statement: Rich people exist.
  • Negation: Rich people do not exist.
  • Equivalent to: Everyone is poor (not rich).
Example: What is the negation of ∀x (x2 > x)?
Answer: ∃x ¬(x2 > x), which is ∃x (x2 ≤ x).

4. Nested Quantifiers

Nested quantifiers occur when one quantifier appears within the scope of another. They are read from left-to-right.

∀x ∃y (x + y = 0)

Read as: "For every real number x, there exists a corresponding real number y such that x + y = 0."

Does Order Matter?

  • Same Quantifiers: Order does not matter.
    ∀x ∀y (x+y = y+x) ≡ ∀y ∀x (x+y = y+x)
  • Mixed Quantifiers: Order changes the meaning completely!

∀x ∃y (x + y = 0)

Meaning: Pick any x you want. I can find a y that makes it zero. (True! If you pick 5, I pick -5).

∃y ∀x (x + y = 0)

Meaning: There is one single magic number y that, when added to ANY number x, equals 0. (False! No single number works for everything).

5. Translating English to Logical Expressions

Example 1: "Every student in this class has studied calculus."
Let C(x) = x has studied calculus. Let S(x) = x is a student in this class.
Translation: ∀x (S(x) → C(x))

Example 2: "Some student in this class has visited Mexico."
Let P(x) = x visited Mexico.
Translation: ∃x (S(x) ∧ P(x))

Complex Translation Example:
Let domain be all students in class.
C(x) = x has a cat, D(x) = x has a dog, F(x) = x has a ferret.
  • "A student in your class has a cat, a dog, and a ferret."
    ∃x (C(x) ∧ D(x) ∧ F(x))
  • "All students in your class have a cat, a dog, or a ferret."
    ∀x (C(x) ∨ D(x) ∨ F(x))
  • "No student in your class has a cat, a dog, and a ferret."
    ∀x ¬(C(x) ∧ D(x) ∧ F(x)) OR ¬∃x (C(x) ∧ D(x) ∧ F(x))

6. Finding Truth Values of Expressions

Determine the truth value of: (∃x (x2 = 2)) → (∀y (y2 ≥ y))

  1. Evaluate the Antecedent (Hypothesis): Is there a real number whose square is 2? Yes, x = √2. So the antecedent is True.
  2. Evaluate the Consequent (Conclusion): Is it true that for ALL real numbers y, y2 ≥ y? Let's test a fraction. If y = 0.5, then 0.52 = 0.25. 0.25 is NOT ≥ 0.5. So the consequent is False.
  3. Final Result: True → False evaluates to False.

Quick Fire True/False (Domain: Real Numbers):

  • ∀x ∃y (x + y = 0) → True
  • ∃y ∀x (x + y = 0) → False
  • ∀x ∃y (xy = 0) → True (Always pick y=0)
  • ∃y ∀x (xy = 0) → True (There is a magic y, y=0, that makes every x multiply to 0)
