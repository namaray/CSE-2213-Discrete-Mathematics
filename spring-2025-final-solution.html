Question 1: Mathematical Induction [3]

Prove: 12 + 32 + 52 + ... + (2n - 1)2 = n(2n - 1)(2n + 1) / 3

Basis Step: Let n = 1.
LHS: (2(1) - 1)2 = 12 = 1.
RHS: 1(2(1) - 1)(2(1) + 1) / 3 = 1(1)(3) / 3 = 1.
Since LHS = RHS, the basis step holds true.
Inductive Hypothesis: Assume the statement is true for n = k, where k ≥ 1.
12 + 32 + ... + (2k - 1)2 = k(2k - 1)(2k + 1) / 3
Inductive Step: Show it is true for n = k + 1.
LHS for n = k+1: [12 + 32 + ... + (2k - 1)2] + (2(k + 1) - 1)2
Substitute the inductive hypothesis:
= [k(2k - 1)(2k + 1) / 3] + (2k + 1)2
= (2k + 1) [ k(2k - 1)/3 + (2k + 1) ]
= (2k + 1) [ (2k2 - k + 6k + 3) / 3 ]
= (2k + 1) (2k2 + 5k + 3) / 3
= (2k + 1) (2k + 3) (k + 1) / 3
= (k + 1) (2(k+1) - 1) (2(k+1) + 1) / 3 = RHS for n = k+1.

By the principle of mathematical induction, the statement is true for all positive integers n.

b) Bit Strings of length 8 [3]

i. At most four 0's:
This means the number of 0's can be 0, 1, 2, 3, or 4. We calculate the combinations C(8, r) for each:
C(8, 0) + C(8, 1) + C(8, 2) + C(8, 3) + C(8, 4) = 1 + 8 + 28 + 56 + 70 = 163 bit strings.

ii. More 0's than 1's:
In an 8-bit string, this requires 5, 6, 7, or 8 zeros.
C(8, 5) + C(8, 6) + C(8, 7) + C(8, 8) = 56 + 28 + 8 + 1 = 93 bit strings.

c) Pigeonhole Principle [2]

Problem: How many people must be in a room to guarantee at least three share the same first letter of their last name (A-Z)?

Solution: There are k = 26 "boxes" (letters). We want at least 3 people in one box.
By the generalized pigeonhole principle, to guarantee ⌈N/26⌉ = 3, the worst-case scenario is having exactly 2 people for every letter (2 × 26 = 52). The very next person (the 53rd) must share a letter with two others.
Therefore, 53 people are required.

Question 3: Graph Theory Basics & Bipartite Graphs [10]

a) Graph Analysis [6]

i. Directed Multigraph True/False: [1]
False. A directed multigraph requires multiple edges between the same two nodes pointing in the same direction. This graph has edges like (1,2) and (2,1), but no two edges are identical in direction. It is a simple directed graph (with a loop).

ii. Handshaking Theorem (Directed Graph): [3]
The directed handshaking theorem states: ∑ deg-(v) = ∑ deg+(v) = |E|.
In-degrees: deg-(1)=0, deg-(2)=4 (from 1,4,5,loop), deg-(3)=2, deg-(4)=1, deg-(5)=1. Sum = 8.
Out-degrees: deg+(1)=2, deg+(2)=1 (loop), deg+(3)=1, deg+(4)=3, deg+(5)=1. Sum = 8.
Total edges |E| = 8. Since 8 = 8 = 8, yes, it mathematically follows the Handshaking Theorem.

b) Bipartite Graph Detection [4]

Problem: Determine if the graph is bipartite using two-coloring algorithm.

Solution: Let's color the graph using Red (R) and Blue (B):
Start with a = R. Its neighbors b, i = B.
Neighbors of b: d, h = R.
Neighbors of i: c, e = R.
Neighbors of d: f = B. (Also neighbors b, c which are B).
Neighbors of e: g = B. (Also neighbors h, i which are B).
Checking remaining edges: f(B) connects to g(R). Wait, tracing carefully gives us sets:
Set 1 (Red): {a, d, h, i, g}
Set 2 (Blue): {b, c, f, e}
Since no two vertices of the same color are adjacent, Yes, the graph is bipartite.

Question 4: Connectivity and Paths [6]

a) Strongly Connected Graph [4]

Problem: Is the graph strongly connected?

Solution: Yes, the graph is strongly connected. A directed graph is strongly connected if there is a directed path from every vertex to every other vertex.
In this graph, node A can reach all other nodes (e.g., A→B→F→H, etc.), and every other node can navigate back to A through the cycle (C→E→G→F→H→B→D→A). Because the entire graph forms one massive interconnected component where returning to the start is always possible, the whole graph is a single strongly connected component.

b) Trolley Routing [2]

i. Weakly Connected: [1]
Yes. A directed graph is weakly connected if replacing all directed edges with undirected edges produces a connected graph. Ignoring edge direction, all 4 sections (R, S, H, K) form a single connected piece.
ii. Path from H to S: [1]
Yes. A book can be sent from History to Science using the path: H → K → R → S.

Question 5: Trees [8]

a) Full 3-ary Tree properties [2]

Total vertices (n) = 100. For a full m-ary tree, n = mi + 1.
100 = 3i + 1 ⇒ 3i = 99 ⇒ i = 33 internal vertices.

i. Number of leaves: L = n - i = 100 - 33 = 67 leaves.
ii. Minimum possible height: L ≤ mh ⇒ 67 ≤ 3h. Since 33=27 and 34=81, the minimum possible height is h = 4.

b) Tree Attributes [2]

i. Depth of the tree: The longest path from the root (A) is A→B→D→G→I (or H). The depth/height is 4.
ii. Ancestor of G: Nodes on the path from root to G: A, B, D.
Descendants of B: All nodes below B: D, E, F, G, I, H.

d) In-order Traversal [3]

Problem: Apply in-order traversal (Left, Root, Right) to the given binary tree. Is it sorted?

Traversal Sequence: 50, 100, 57, 57, 21, 200, 300.
Is it sorted? No. It is not sorted because the given tree is an arbitrary binary tree, not a valid Binary Search Tree (BST). For example, 21 is the right child of 100 (but 21 < 100), meaning an in-order traversal will inherently output numbers out of numerical sequence.
