GUARANTEED Must-Know Concepts

These have appeared in every single final exam paper. Do not walk in without mastering these.

1. Bipartite Graph (2-Coloring Algorithm)

Definition: A graph is bipartite if its vertices can be split into two disjoint groups V1 and V2 such that every edge connects a vertex in V1 to one in V2. No edges exist within the same group.

The Algorithm (always the same steps):
  1. Pick any vertex and color it Red.
  2. Color all its neighbors Blue.
  3. Color all their neighbors Red, and so on.
  4. If you ever find two adjacent vertices of the same color, the graph is NOT bipartite.
  5. If the entire graph is colored without conflict, it IS bipartite. State the two sets.

2. Identifying Graph Type from a Scenario

Every paper has a word problem where you must state the correct graph type and justify it.

The 3 Key Questions to Ask:
1. Directed or Undirected? → Is the relationship one-way (e.g., "calls", "beats") or mutual (e.g., "knows", "connected to")?
2. Loops? → Can a vertex connect to itself (e.g., a drone returning to its base)?
3. Multiple Edges? → Can two vertices be connected by more than one edge (e.g., multiple train lines between two stations)?

LIKELY High-Probability Topics

3. Binary Search Tree (BST) Construction

Rule: Left child < parent < right child (use dictionary order for words).

Step-by-step: Insert the first word as the root. For each subsequent word, start at the root and go LEFT if smaller, RIGHT if larger, until you find an empty spot.

BST Traversal sequences (know all 3):

  • Pre-order → Root, Left, Right
  • In-order → Left, Root, Right   (produces a sorted sequence in a valid BST)
  • Post-order → Left, Right, Root

4. Full m-ary Tree Formulas

Given m (branching factor) and i (internal vertices) or total vertices n:
  • Total vertices: n = m·i + 1
  • Number of leaves: l = (m-1)·i + 1
  • Max leaves at height h: Lmax = mh
  • Min height for l leaves: h = ⌈logm(l)⌉

5. Handshaking Theorem

Undirected: ∑ deg(v) = 2|E|
Directed: ∑ deg-(v) = ∑ deg+(v) = |E|

Common question formats:

  • "Find the missing degree d given 9 other degrees and total edges." → Sum all given degrees, subtract from 2×|E|.
  • "Does this graph follow the Handshaking Theorem?" → Sum all degrees and check if divisible by 2.
  • "Find the number of edges in Kn (complete graph)." → Each vertex has degree n-1, so 2|E| = n(n-1), thus |E| = n(n-1)/2.

6. Pigeonhole Principle

To guarantee at least k objects in one box (with n boxes), you need: N = n(k-1) + 1 objects.

Common question formats:

  • "How many students to guarantee 9 from the same batch (5 batches)?" → 5(9-1)+1 = 41
  • "How many people to guarantee 4 pick the same genre (10 genres)?" → 10(4-1)+1 = 31
  • "Min books so one shelf has ≥15 (10 shelves)?" → 10(15-1)+1 = 141

7. Combinations with Conditions ("at least" / "at most")

Formula: C(n, r) = n! / (r! × (n-r)!)
"At least k of type A" → Sum all valid cases: k of A, k+1 of A, etc.
"At least 1 of type A" → Total - (combinations with NONE of type A)

GUARANTEED FROM MIDTERMS Functions — Now on Final Syllabus

Functions appeared in every single midterm paper as Q4, always in two parts. Since it has now moved from the midterm to the final syllabus, you must treat it as a guaranteed final topic.

Part A: Is this a function? / Composite Functions

Is this a function? A relation is a function if every input maps to exactly one output.

Common "Is this a function?" traps:
  • g(x) = ±√x → NOT a function. One input (e.g., x=4) gives two outputs (+2 and -2).
  • f(x) = 1/x → NOT a function on ℝ, because x=0 is undefined. It IS a function on ℝ−{0}.
  • f(x) = 2x+3/(6x-2) → Check if denominator can be zero. 6x-2=0 gives x=1/3, so the domain must exclude x=1/3.

Composite Functions: (f∘g)(x) = f(g(x)) — substitute g(x) into f.

Example: If f(x) = x²-5x+1 and g(x) = 7x-3:
(f∘g)(x) = f(7x-3) = (7x-3)²-5(7x-3)+1 = 49x²-42x+9-35x+15+1 = 49x²-77x+25

Part B: Show the function is invertible & find its inverse

This is a 4-mark question in every paper. It always has the same structure: a rational function of the form f(x) = (ax+b)/(cx+d).

The exact 3-step method used in every paper:
  1. Prove it is one-to-one: Assume f(x₁) = f(x₂). Cross-multiply and simplify to get x₁ = x₂. Done.
  2. Prove it is onto: Let y ∈ codomain. Solve for x in terms of y. Show x is in the domain. Done.
  3. Find the inverse: Replace f(x) with y → swap x and y → solve for y → that is f⁻¹(x).
Worked Example (Paper 252): Show f(x) = (x+1)/(2x-1), f : ℝ-{½} → ℝ-{1} is invertible, find f⁻¹(x), evaluate f⁻¹(2).

One-to-one: f(x₁) = f(x₂) → (x₁+1)/(2x₁-1) = (x₂+1)/(2x₂-1). Cross-multiply → x₁ = x₂. ✓
Onto: Let y = (x+1)/(2x-1). Solve: y(2x-1) = x+1 → 2xy-y = x+1 → x(2y-1) = y+1 → x = (y+1)/(2y-1). This is real for all y ≠ ½ ∈ codomain. ✓
Inverse: f⁻¹(x) = (x+1)/(2x-1) (same formula here — this is a self-inverse function).
Evaluate: f⁻¹(2) = (2+1)/(2(2)-1) = 3/3 = 1.

Key Function Definitions

  • One-to-one (Injective): f(x₁) = f(x₂) implies x₁ = x₂. No two inputs share the same output.
  • Onto (Surjective): Every element of the codomain has at least one pre-image. Range = Codomain.
  • Bijection: Both one-to-one AND onto. Required for invertibility.
  • Invertible: A bijection f has an inverse f⁻¹ such that f⁻¹(f(x)) = x for all x in the domain.

POSSIBLE Know These Definitions Cold

8. Tree Attributes

  • Height/Depth: Number of edges on the longest path from root to any leaf.
  • Level of a node: Number of edges from the root to that node.
  • Ancestor of node X: All nodes on the path from the root to X (not including X).
  • Descendants of node X: All nodes reachable from X going downward (not including X).
  • Leaf: A node with no children.
  • Internal vertex: A node with at least one child.

9. Graph Representations

  • Adjacency Matrix: Square matrix. Row = source vertex, Column = destination. Cell = 1 if edge exists.
  • Incidence Matrix: Rows = vertices, Columns = edges. Cell = 1 if the vertex is an endpoint of that edge.

10. Strongly Connected vs Weakly Connected

  • Strongly Connected: There is a directed path from EVERY vertex to EVERY other vertex.
  • Weakly Connected: Ignoring edge direction, the underlying graph is connected.
  • Strongly Connected Component (SCC): A maximal subgraph that is strongly connected.

11. Graph Isomorphism

Two graphs are isomorphic if one can be redrawn to look identical to the other (relabeling vertices).

To prove NOT isomorphic, check these invariants in order:
  1. Number of vertices must be equal.
  2. Number of edges must be equal.
  3. Degree sequence (sorted list of all vertex degrees) must be identical.
If any one of these fails, they are NOT isomorphic.

12. Mathematical Induction (Proof Template)

  1. Basis Step: Prove the formula holds for n=1 (show LHS = RHS).
  2. Inductive Hypothesis: Assume the formula holds for n=k.
  3. Inductive Step: Show it holds for n=k+1 by substituting the hypothesis and simplifying to match the formula for k+1.
  4. Conclusion: "By the Principle of Mathematical Induction, the statement is true for all positive integers n."
