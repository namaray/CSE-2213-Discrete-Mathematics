1. Introduction to Graphs

A graph G = (V, E) consists of V, a nonempty set of vertices (or nodes) and E, a set of edges. It shows connections between things.

Graph Examples

  • Acquaintanceship Graph: Undirected (if Alice knows Bob, Bob knows Alice).
  • Call Graph: Directed (Alice calls Bob).
  • Hollywood Graph: Simple undirected graph (Actors working in a movie together).
Handshaking Theorem (Undirected):
For any undirected graph with e edges, the sum of the degrees of the vertices is 2e.
∑ deg(v) = 2e
Handshaking Theorem (Directed):
∑ deg-(v) = ∑ deg+(v) = |E|

2. Special Graph Structures

Complete Graphs (Kn)

Every pair of distinct vertices is connected by a unique edge. Edges = n(n-1)/2.

Cycles (Cn)

A path starting and ending at the same vertex. Edges = n.

Wheels (Wn)

Formed by connecting a single central vertex to all vertices of a cycle. Edges = 2n.

n-Cubes (Qn)

Vertices represent 2n bit strings of length n. Adjacent if strings differ by exactly 1 bit.

Bipartite Graphs:
A simple graph is bipartite if its vertex set can be partitioned into two disjoint sets V1 and V2 such that every edge connects a vertex in V1 to one in V2. No edges exist within the same group. Complete Bipartite is Km,n.

3. Graph Isomorphism

We can draw the same graph differently. If two graphs are actually two different representations of the same graph, they are isomorphic.

Determining isomorphism is hard (n! possibilities). It is often easier to prove they are NOT isomorphic by checking invariants:

  • Number of vertices must match.
  • Number of edges must match.
  • Degree Sequence: The sorted list of vertex degrees must be identical.

4. Graph Representations

1. Adjacency List

A table mapping each vertex to its adjacent vertices (or terminal vertices for directed graphs).

2. Adjacency Matrix

A square matrix A where Aij = 1 if there is an edge from vi to vj, and 0 otherwise.

3. Incidence Matrix

A matrix where rows represent vertices and columns represent edges.
Mij = 1 if edge ej is incident with vertex vi, and 0 otherwise.

5. Paths and Connectivity

A path is a sequence of edges. A simple path does not repeat any edges. A circuit begins and ends at the same vertex.

  • Connected (Undirected): There is a path between every pair of vertices.
  • Strongly Connected (Directed): There is a directed path from a to b and from b to a for every pair of vertices.
  • Weakly Connected (Directed): The underlying undirected graph is connected.
  • Strongly Connected Components: Maximal subgraphs that are strongly connected.
